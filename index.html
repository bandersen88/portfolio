Handyman Plan

Home Repair Made Simple

Handyman Plan solves the problem of coordinating and completing unexpected home maintence on a budget. Inspired by the neighborhood user group model of Nextdoor, Handymanplan first connects users with neighborhoods who may be able and willing to help with their home repairs, with the option always to schedule a contractor and schedule professional repairs.

Users sign up, linking them by their address to their neighborhood, and submit tickets for home repair projects, which is covered under their monthly insurance premium plan plus copay. The end result is convenient home maintence with predicatable cost and net cost savings

In summary, what we built is:

  • Express server for handling all requests
  • MySQL database
  • Sequelize ORM for building relationships among the db tables and crud operations
  • Passport.js authentication and password encryption
  • Bootstrap front end

Polygon Scraper

View and comment on the latest articles from Polygon

This site scrapes Polygon.com and provides a list of all articles on the front page.

All articles from the front page of Polygon are shown. Users can click the "get new articles" button to update the list of articles, and they can go to an article's specific page to leave comments on it.

In summary, what I built is:

  • A MongoDB database to hold all article information and any comments on each article
  • An Express server that uses the Cheerio NPM module to scrape all articles from the front page of Polygon and pass their data to Mongo and Handlebars
  • A Handlebars-powered frontend that takes the article information it's passed and displays it as HTML

jQuery RPG

Defeat all of your enemies before they defeat you

This is an RPG built using only jQuery and plain JavaScript, based on the Atlus video game series "Shin Megami Tensei: Persona".

The player chooses one of six characters, each with different stats, and then must defeat all five other characters without losing all of their health. The player character's attack stat increases on each attack, and they choose the order in which to challenge the other characters. The player must choose the order of opponents carefully in order to survive long enough to defeat every opponent. The game's battle music changes depending on the character the player chooses.

In summary, what I built is:

  • Dynamically-generated HTML, using jQuery to build each character's info card and update it when necessary
  • Main game object that handles progression of game, including logic for dealing damage, increasing stats, and defeating enemies
  • jQuery event handlers for selecting characters and which trigger relevant game-logic functions in the main game object
