West Indigo

Ecommerce Website Layout

West Indigo is a sample eCommerce website built with the MERN tech stack.

The site makes use of a large Mongo database of items in 5 different categories, which can be searched and filtered and commented on.

In summary, what I built is:

  • A React-based frontend with a front page, a Search page, and a Product page that shows information on one item
  • A MongoDB database that holds all the information on each product as well as comments on each product
  • An Node/Express backend to run the API server, which handles all queries between React and Mongo such as CRUD functions interacting with the product database
  • CSS compiled using the LESS preprocessor

MyMovieList

Track, rate, and share the movies you've seen.

MyMovieList is a single-page web app that allows users to list and rate every movie they've seen, and allows them to share with their friends or online.

Built from scratch by myself and four others over 3 weeks, the site facilitates searching for movies to add to your list, rating them from 1-10, and marking them as "Completed", "Dropped", or "Plan to Watch". Users can also go to a specific movie's page for more detailed information and trailers, and look up and view other user's lists. I built most of the functionality of the List and Search pages, including most of the React code on those pages as well as the API server and backend routing for most of the application.

In summary, what we built is:

  • A React-based frontend with a home view, a view that displays a user's movie list, a search view that queries the OMDB API for movies to add to a user's list, and a view with detailed information on a specific movie
  • A MongoDB database that holds each user's profile information as well as their personal movie list
  • An Node/Express backend to run the API server, which handles all queries between React and Mongo such as CRUD functions interacting with a user's movie list
  • A login authentication system using the Passport middleware for Express
  • Bootstrap- and Materialize-powered CSS styling

Polygon Scraper

View and comment on the latest articles from Polygon

This site scrapes Polygon.com and provides a list of all articles on the front page.

All articles from the front page of Polygon are shown. Users can click the "get new articles" button to update the list of articles, and they can go to an article's specific page to leave comments on it.

In summary, what I built is:

  • A MongoDB database to hold all article information and any comments on each article
  • An Express server that uses the Cheerio NPM module to scrape all articles from the front page of Polygon and pass their data to Mongo and Handlebars
  • A Handlebars-powered frontend that takes the article information it's passed and displays it as HTML

React Memory Game

Don't click on the same image twice

This is a memory game built entirely with React, using images from the Capcom game "Monster Hunter World".

The player gets one point for clicking on an image, but if they click on the same image twice they lose and the game resets. The order of the images reshuffles after each click.

In summary, what I built is:

  • React components for the clickable images and the game header
  • A main class component that holds an array of every image that shuffles on every click, an array of all the images that the player has clicked so far, and the player's current score and highest score

jQuery RPG

Defeat all of your enemies before they defeat you

This is an RPG built using only jQuery and plain JavaScript, based on the Atlus video game series "Shin Megami Tensei: Persona".

The player chooses one of six characters, each with different stats, and then must defeat all five other characters without losing all of their health. The player character's attack stat increases on each attack, and they choose the order in which to challenge the other characters. The player must choose the order of opponents carefully in order to survive long enough to defeat every opponent. The game's battle music changes depending on the character the player chooses.

In summary, what I built is:

  • Dynamically-generated HTML, using jQuery to build each character's info card and update it when necessary
  • Main game object that handles progression of game, including logic for dealing damage, increasing stats, and defeating enemies
  • jQuery event handlers for selecting characters and which trigger relevant game-logic functions in the main game object
